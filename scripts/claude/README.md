# Claude Synthetic Data Training Configurations

This directory contains configuration files for NER model training using synthetic data generated by Claude.

## Configuration Files

| Config File | Description |
|-------------|-------------|
| `config_two_stage_slm_claude.json` | Two-stage training template (use `--topn` to specify N) |
| `config_three_stage_slm_claude.json` | Three-stage training template (use `--topn` to specify N) |

Both files use `${topn}` as a template variable. Pass `--topn <value>` to `run.sh` to select the synthetic dataset (topn5, topn10, topn20, topn50, topn100).

### Dataset Paths

Config files reference two datasets in `data.datasets`:
1. **Synthetic data**: `dataset/yakusha_synthetic_topn${topn}_uniform_claude` -- Claude generated NER training data, `${topn}` is replaced by the `--topn` argument
2. **Few-shot data**: `dataset/yakusha_annotated_data_few-shot` -- Human-annotated real data for fine-tuning

### Two-Stage Training

Two-stage training pipeline:
1. Pre-train NER using Claude synthetic data
2. Fine-tune on few-shot real data

### Three-Stage Training

Three-stage training pipeline:
1. MLM pre-training (corpus_gennbun.txt, 100000 lines)
2. Train NER using Claude synthetic data
3. Fine-tune on few-shot real data

## Included Models

All configurations include the following 14 models:
- BERT / BERT+CRF
- BERT-Large / BERT-Large+CRF
- RoBERTa / RoBERTa+CRF
- LUKE / LUKE+CRF
- ALBERT / ALBERT+CRF
- DeBERTa / DeBERTa+CRF
- BiLSTM / BiLSTM+CRF

## Execution Commands

Run from the project root directory `/root/Yakushahyoubanki_NER`:

### Run All TopN Experiments (Recommended)

```bash
cd /root/Yakushahyoubanki_NER && source .venv/bin/activate && nohup bash -c '
for topn in 5 10 20 50 100; do
    echo "========================================"
    echo "Running: two-stage claude topn=$topn"
    echo "========================================"
    ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn $topn
done
echo "All claude two-stage experiments completed!"
' > logs/claude_two_stage_all.log 2>&1 &
```

### Run Individual Experiments

#### Two-Stage Training

```bash
nohup ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn 5 > logs/two_stage_slm_top5_claude.log 2>&1 &
nohup ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn 10 > logs/two_stage_slm_top10_claude.log 2>&1 &
nohup ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn 20 > logs/two_stage_slm_top20_claude.log 2>&1 &
nohup ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn 50 > logs/two_stage_slm_top50_claude.log 2>&1 &
nohup ./scripts/run.sh --config scripts/claude/config_two_stage_slm_claude.json --topn 100 > logs/two_stage_slm_top100_claude.log 2>&1 &
```

nohup ./scripts/run.sh --config scripts/claude/config_three_stage_slm_claude.json --topn 20 > logs/three_stage_slm_top20_claude.log 2>&1 &
nohup ./scripts/run.sh --config scripts/claude/config_three_stage_slm_claude.json --topn 50 > logs/three_stage_slm_top50_claude.log 2>&1 &

> GPU is auto-scheduled: `run.sh` detects free GPUs and assigns tasks automatically. Use `--gpu N` to force a specific GPU.

## Output Directory

Experiment results are saved in the `experiments/` directory with the following format:
- Two-Stage: `experiments/two_stage_slm_top{N}_claude_${timestamp}/`
- Three-Stage: `experiments/three_stage_slm_top{N}_100000_claude_${timestamp}/`

Note: The `_claude` suffix distinguishes these from ChatGPT experiments.
